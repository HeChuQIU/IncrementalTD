# Research: 科幻极简视觉风格实现方案

## 1. Phaser 3 中实现硬朗几何与护甲质感

**Decision**: 使用 `Phaser.GameObjects.Graphics` 绘制复杂的几何图形，然后通过 `generateTexture()` 将其转换为可复用的 Sprite 纹理。

**Rationale**: 
- **性能**: 相比于每帧重新绘制复杂的 Graphics 路径，使用预生成的纹理（Sprite）渲染效率极高，能够轻松满足 50+ 敌人同屏的性能要求。
- **极简风格**: 极简风格不需要复杂的位图纹理，纯色填充、渐变和几何线条足以表现"硬朗几何"和"护甲质感"（例如通过叠加不同深浅的灰色多边形来模拟装甲板）。
- **无外部依赖**: 避免引入外部图片资源，保持项目的轻量化和纯代码驱动。

**Alternatives considered**: 
- *直接使用 Graphics 对象*: 对于简单的矩形/圆形可行，但对于带有护甲细节的多边形，大量实例会导致性能下降。
- *引入外部 Sprite 图片*: 增加了美术资源管理的负担，不符合当前纯代码驱动的开发模式。

## 2. Phaser 3 中的发光/泛光（Glow/Bloom）效果

**Decision**: 优先使用 `blendMode: Phaser.BlendModes.ADD` 配合半透明的几何图形来模拟发光效果；对于关键元素（如子弹、选中的塔），谨慎使用 Phaser 3.60+ 内置的 `postFX.addGlow()`。

**Rationale**: 
- **性能优先**: 叠加混合模式（ADD）是实现发光效果性能开销最小的方式，非常适合大量子弹和敌人的场景。
- **视觉效果**: 在深色背景下，ADD 混合模式能产生极佳的霓虹/科幻发光感。
- **内置特效**: `postFX` 虽然效果更好，但在大量对象上使用会导致严重的性能下降，因此仅限于少量高亮元素。

**Alternatives considered**: 
- *全局 Bloom 后期处理*: 性能开销过大，可能导致低端设备掉帧，违背了维持 95% 帧率的性能目标。

## 3. 中性平衡配色方案规范

**Decision**: 采用基于 Tailwind CSS 色板的深蓝灰（Slate/Cyan/Orange）配色方案。

**Rationale**: 
- **背景与环境**: 使用深蓝灰（Slate 900: `#0F172A`）作为主背景，Slate 700 (`#334155`) 作为网格线，营造深邃的太空/科技感，且不刺眼。
- **玩家元素（塔、UI主色）**: 使用青色/青蓝色（Cyan 400: `#22D3EE` / Cyan 500: `#06B6D4`），代表友方、科技、能量。
- **敌方元素**: 使用橙色/红色（Orange 500: `#F97316` / Red 500: `#EF4444`），提供强烈的视觉对比，符合"战术色调"和警告意味。
- **中性护甲**: 使用不同层级的灰色（Slate 400 到 Slate 600）来绘制塔和敌人的主体装甲。
- **色盲友好**: 青色（蓝绿）与橙色/红色的对比在大多数色盲类型（包括红绿色盲）中都能保持清晰的区分度。

**Alternatives considered**: 
- *纯黑白+单一点缀色*: 过于单调，难以区分不同类型的敌人和塔。
- *高饱和度赛博朋克（紫/品红/亮黄）*: 视觉刺激过强，长时间游玩容易疲劳，不符合"中性平衡"的要求。
